# Supreme Council & Deliberation Protocol

Every output generated by Aetherion passes through a rigorous peer‑review by the 7‑judge **Supreme Council**.

## Council Composition

| Judge | Mandate |
|-------|---------|
| **Critic** | Finds the strongest argument against the output. |
| **Security** | Identifies vulnerabilities, unsafe patterns, or exposed secrets. Holds **absolute veto**. |
| **Alignment** | Checks whether the output matches the original user request. |
| **Constraint** | Verifies scope and resource limits. |
| **Evaluator** | Scores overall quality 0–10. |
| **Documentation** | Assesses readability and clarity. |
| **AetherionPrime** | Acts as tiebreaker and final arbiter. |

## Pre‑Council Pipeline

1. **Sanitizer** – Removes markdown formatting and conversational filler.
2. **Forensic Analyst** – Fact‑checks imports, API existence, resource claims.
3. **Edge‑Case Generator** – Produces adversarial inputs to test robustness.

## Voting Protocol

Each judge returns a JSON vote with:
- `verdict`: `"approve"`, `"reject"`, `"revision"`, or `"abstain"`
- `confidence`: 0.0 to 1.0
- `score`: 0.0 to 10.0
- `reasoning`: textual explanation

### Security Veto

If **Security** votes `REJECT` or `REVISION`, the entire output is **REJECTED** with score 0.0, regardless of other votes. This is an absolute, non‑negotiable safety boundary.

### Weighted Voting

Each judge has a reputation weight ($w_j$) initialised at 1.0 and updated after every task based on the Council's overall score. Weights are clamped between 0.7 and 1.5.

The weighted average score is:
$$\bar{s} = \frac{\sum_{j \in J} w_j \cdot s_j}{\sum_{j \in J} w_j}$$

### Final Verdict

Configurable thresholds (defaults):
- $\bar{s} \ge 7.0$ → **APPROVED**
- $5.0 \le \bar{s} < 7.0$ → **REVISION REQUIRED**
- $\bar{s} < 5.0$ → **REJECTED**

These thresholds can be customised per workspace via the **Constitution Editor**.

## Bias Detection

The **Juror** agent checks for:
- **Groupthink** – all scores within 0.5 of the mean.
- **Unanimous Approval Bias** – all judges approve without critical scrutiny.
- **Anchoring, complexity bias, and confirmation bias** via LLM analysis.

## Reputation Feedback

After each task, the Council score auto‑updates judge reputations:
- Score ≥ 7.0 → improved
- Score ≤ 4.0 → decreased

This creates a self‑improving governance layer over time.

## Constitution Editor

Enterprise workspaces can customise:
- Judge prompts
- Voting thresholds
- Enable/disable individual judges

All changes are audit‑logged per workspace.
